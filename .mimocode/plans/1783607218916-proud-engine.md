# Portfolio Feature Expansion Plan

## Overview
Add 12 new features to Kian Janloo's portfolio site (React 19 + TypeScript + Vite + Tailwind v4 + Framer Motion). Two originally suggested features are skipped (GitHub Graph — user declined; Dark Mode Auto-Detect & Unified Timeline — already exist).

---

## Feature 1: Blog / Articles Section
**New files:** `src/components/landing/blog/Blog.tsx`, `BlogPost.tsx`, `BlogModal.tsx`, `src/utils/service/blog/blog.ts`
**Modify:** `src/components/landing/Landing.tsx` (add blog section + nav item)

- Create a `blog` data file with sample posts as TypeScript objects (title, date, excerpt, tags, content as HTML string, readTime)
- `Blog.tsx`: grid of blog post cards (2-col, same pattern as Projects)
- `BlogPost.tsx`: card with title, date, read time, tags, excerpt. Click opens `BlogModal.tsx`
- `BlogModal.tsx`: full article view in modal (AnimatePresence, same pattern as ProjectModal). Renders `content` via dangerouslySetInnerHTML
- Add "Blog" nav item and section between Projects and Contact
- Section heading uses `<FlipText>` like other sections

---

## Feature 2: Testimonials / Recommendations
**New files:** `src/components/landing/testimonials/Testimonials.tsx`, `TestimonialCard.tsx`, `src/utils/service/testimonials/testimonials.ts`
**Modify:** `src/components/landing/Landing.tsx`

- Testimonials data: name, role, company, quote, avatar (use initials as fallback)
- `Testimonials.tsx`: horizontal scroll carousel or grid
- `TestimonialCard.tsx`: glass card with quote icon, text, attribution, star rating
- Add section between Experience and Projects

## Feature 3: Interactive Project Demos
**Modify:** `src/components/landing/projects/Project.tsx`, `ProjectModal.tsx`, `src/utils/service/projects/projects.ts`

- Add `demoUrl` field to project data for projects with live demos
- In ProjectModal, embed an iframe preview when `demoUrl` exists
- Add a "Live Demo" button that opens an embedded preview panel within the modal
- Iframe with loading skeleton, max-height, rounded corners

## Feature 4: i18n (Farsi/English)
**New files:** `src/i18n/translations.ts`, `src/i18n/LanguageContext.tsx`, `src/components/common/LanguageSwitcher.tsx`
**Modify:** All component files that render text (Summary, Experience, Projects, Contact, Footer, Landing nav)

- Create `translations.ts` with all English keys as defaults + empty Farsi placeholders (user will provide translations later)
- `LanguageContext.tsx`: provides `language` state + `toggleLanguage()` + `t(key)` function
- `LanguageSwitcher.tsx`: small toggle button (EN/FA) next to ThemeSwitcher in nav
- Experience/project data gets `titleFa`/`bulletsFa` optional fields (initially same as English)
- Summary bio text gets translation variants
- Persist language choice to localStorage
- RTL support: add `dir="rtl"` to html when Farsi is selected, adjust CSS accordingly

---

## Feature 5: Visitor Counter
**New files:** `src/components/common/VisitorCounter.tsx`
**Modify:** `src/components/common/Footer.tsx`

- Simple client-side counter using localStorage (increment on each visit with a unique session key)
- Display in footer: "Visitor #1,234"
- Glass pill styling, small and unobtrusive
- Animated number roll-up on load

---

## Feature 6: Command Palette (Ctrl+K)
**New files:** `src/components/common/CommandPalette.tsx`, `src/hooks/useCommandPalette.ts`
**Modify:** `src/app/App.tsx`

- Overlay modal triggered by Ctrl+K / Cmd+K
- Search through: sections (nav items), projects, skills, social links, actions (download resume, toggle theme, toggle language)
- Fuzzy search with highlighted matches
- Keyboard navigation: arrow keys to select, Enter to activate, Escape to close
- Glass morphism overlay with backdrop blur
- `useCommandPalette` hook manages open/close state and global keydown listener

## Feature 13: Keyboard Shortcuts
**New files:** `src/hooks/useKeyboardShortcuts.ts`
**Modify:** `src/app/App.tsx`

- Global keydown listener for:
  - `j` / `k`: scroll to next/prev section
  - `t`: toggle theme
  - `d`: download resume
  - `/` or `Ctrl+K`: open command palette
  - `1-5`: jump to specific sections
- Visual hint: small toast notification showing shortcut was triggered
- Respect `prefers-reduced-motion` and don't override when input is focused

## Implementation Order

### Batch 1 — Independent foundations
1. **i18n** (Feature 4) — needed by many other features
2. **Keyboard shortcuts** (Feature 13) — standalone hook
3. **Command palette** (Feature 6) — standalone component
4. **PWA** (Feature 9) — standalone config

### Batch 2 — New sections (parallelizable)
5. **Blog** (Feature 1)
6. **Testimonials** (Feature 2)
7. **Currently Learning** (Feature 12)
8. **Radar Chart** (Feature 7)

### Batch 3 — Enhancements
9. **Interactive project demos** (Feature 3)
10. **Visitor counter** (Feature 5)
11. **Animated page transitions** (Feature 8)

### Batch 4 — Build tools
12. **RSS feed** (Feature 10)
13. **OG image** (Feature 11)
